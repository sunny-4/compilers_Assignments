Language Specification
(C-like Language Subset)
Group ID: 22
1 Introduction
Programming languages are processed using multiple stages such as lexical analysis, syntax analysis, semantic analysis, and code generation. Among these stages, lexical analysis
is the first and one of the most important steps in the compilation or interpretation process. The lexical analyzer reads the source program as a sequence of characters and
groups them into meaningful units called tokens.
This report presents the specification of a simplified C-like programming language
designed specifically for implementing a lexical analyzer using tools such as Lex or Flex.
The goal of this specification is to clearly define all token types supported by the language
so that the lexical analyzer can correctly identify and classify them.
The language defined in this document is a restricted subset of ANSI C. The restriction
is intentional in order to simplify implementation while still maintaining similarity with
real-world programming languages.
2 Keywords
Keywords are reserved words that have predefined meanings in the programming language
and cannot be used as identifiers.
The lexical analyzer must recognize the following keywords:
int , long , short , float , double , void , char ,
struct , extern ,
if , else , switch , case , default ,
for , while , do ,
break , continue , return , sizeof
Note: All other C keywords not listed above must be treated as identifiers.

3 Identifiers
Definition: Identifiers are user-defined names used for variables, functions, and structures.
Rules:
• Must start with a letter or underscore
• Can contain letters, digits, and underscores
• Cannot be a keyword
Pattern:
Letter ( Letter | Digit | _ ) *
Regular Expression:
[A - Za - z_ ][ A - Za - z0 -9 _ ]*

12 Conclusion
This document defines a simplified yet practical language specification suitable for implementing a lexical analyzer using Lex/Flex. The specification focuses on accurate token
recognition while avoiding unnecessary complexity from full C language support. The language includes major programming constructs such as identifiers, constants, operators,
and comments, which are sufficient for understanding lexical analysis concepts.
